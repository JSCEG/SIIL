Recuperación controlada por correo institucional

Este flujo usa el enlace firmado de Supabase y evita exponer credenciales administrativas por medios informales.

  • El correo de recuperación se envía únicamente a la dirección registrada en la cuenta.
  • El enlace debe abrirse directamente desde el mensaje recibido para validar el token.
  • Después del cambio, la sesión anterior se invalida y se requiere iniciar sesión nuevamente.
